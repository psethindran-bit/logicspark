LED Blink & GPIO Control
Control microcontroller GPIO pins to blink LEDs and drive basic outputs. Essential first step for hardware programming.
  • Configure pins as INPUT/OUTPUT using pinMode()
  • Digital output control with digitalWrite(HIGH/LOW)
  • Timing control using delay() and millis()
  • Understand microcontroller pin specifications
  • Calculate resistor values for safe current limiting
const int LED_PIN = 13;
const int DELAY_MS = 500;

void setup() {
  pinMode(LED_PIN, OUTPUT);  // Configure pin as output
  Serial.begin(9600);        // Initialize serial monitor
}

void loop() {
  digitalWrite(LED_PIN, HIGH);   // Turn LED ON
  Serial.println("LED ON");
  delay(DELAY_MS);
  
  digitalWrite(LED_PIN, LOW);    // Turn LED OFF
  Serial.println("LED OFF");
  delay(DELAY_MS);
}
Pro Insight: Use 220-330Î© resistors for LEDs. Always check microcontroller datasheet for max current per pin (typically 20mA).
BEGINNER
Digital Inputs & Debouncing
Read button presses and switch states with proper debouncing to prevent false triggers.
  • Read digital inputs using digitalRead()
  • Implement software debouncing techniques
  • Use internal/external pull-up resistors
  • Create reactive systems to user input
  • Handle multiple input sources
const int BUTTON_PIN = 2;
const int LED_PIN = 13;
bool lastButtonState = HIGH;
unsigned long lastDebounceTime = 0;
const unsigned long DEBOUNCE_DELAY = 50;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  bool reading = digitalRead(BUTTON_PIN);
  
  if (reading != lastButtonState) {
    lastDebounceTime = millis();
  }
  
  if ((millis() - lastDebounceTime) > DEBOUNCE_DELAY) {
    if (reading == LOW) {
      digitalWrite(LED_PIN, HIGH);  // Button pressed
    } else {
      digitalWrite(LED_PIN, LOW);   // Button released
    }
  }
  
  lastButtonState = reading;
}
Pro Insight: 10kÎ© pull-up resistors prevent floating inputs. Debounce delay of 50ms works for most mechanical switches.

INTERMEDIATE
PWM & Analog Sensors
Control brightness, motor speed, and read analog sensors like potentiometers and LDRs.
  • Generate PWM signals using analogWrite()
  • Read analog inputs with analogRead() (0-1023)
  • Map sensor values to control parameters
  • Filter noisy analog readings
  • Drive servos, LEDs, and DC motors
const int POT_PIN = A0;
const int LED_PIN = 9;
const int LDR_PIN = A1;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  int potValue = analogRead(POT_PIN);      // 0-1023
  int ldrValue = analogRead(LDR_PIN);      // Light sensor
  
  // Map potentiometer to PWM (0-255)
  int brightness = map(potValue, 0, 1023, 0, 255);
  analogWrite(LED_PIN, brightness);
  
  Serial.print("Pot: "); Serial.print(potValue);
  Serial.print(" | LDR: "); Serial.print(ldrValue);
  Serial.print(" | PWM: "); Serial.println(brightness);
  
  delay(100);
}
Pro Insight: PWM frequency ~490Hz on most Arduino pins. Use map() function for clean value scaling.
INTERMEDIATE
Timers & Non-Blocking Code
Replace delay() with millis() for responsive, multi-tasking applications.
  • Non-blocking timing using millis()
  • Multiple independent timed events
  • State machines for complex behaviors
  • Hardware/software timers
  • Avoid blocking the main loop
const int LED1_PIN = 13;
const int LED2_PIN = 12;
unsigned long previousMillis1 = 0;
unsigned long previousMillis2 = 0;
const long interval1 = 1000;
const long interval2 = 1500;
bool led1State = false;
bool led2State = false;

void setup() {
  pinMode(LED1_PIN, OUTPUT);
  pinMode(LED2_PIN, OUTPUT);
}

void loop() {
  unsigned long currentMillis = millis();
  
  // LED 1 - 1 second interval
  if (currentMillis - previousMillis1 >= interval1) {
    previousMillis1 = currentMillis;
    led1State = !ledState;
    digitalWrite(LED1_PIN, led1State);
  }
  
  // LED 2 - 1.5 second interval (runs simultaneously)
  if (currentMillis - previousMillis2 >= interval2) {
    previousMillis2 = currentMillis;
    led2State = !led2State;
    digitalWrite(LED2_PIN, led2State);
  }
}
Pro Insight: millis() overflows after ~49 days but works fine (Arduino unsigned long math handles it).

Serial Communication & Protocols
Master UART, I2C, SPI communication between devices and sensors.

#include 
#define I2C_SLAVE_ADDRESS 0x04

void setup() {
  Wire.begin(I2C_SLAVE_ADDRESS);  // Start I2C as slave
  Wire.onReceive(receiveEvent);
  Wire.onRequest(requestEvent);
  Serial.begin(9600);
}

void loop() {
  delay(100);
}

// Called when data received from master
void receiveEvent(int bytes) {
  while (Wire.available()) {
    char c = Wire.read();
    Serial.print("Received: ");
    Serial.println(c);
  }
}

// Called when master requests data
void requestEvent() {
  Wire.write("Hello Master!");
}
Pro Insight: I2C addresses: 0x03-0x77. Use pull-up resistors (4.7kÎ©) on SDA/SCL lines.

ADVANCED
Interrupts & Real-Time Control
Handle hardware events instantly with interrupt service routines (ISRs).
  • External pin interrupts (RISING/FALLING)
  • Timer interrupts for precise timing
  • ISR best practices (keep short, no delay())
  • Interrupt priority management
  • Encoder reading and motor control
volatile bool buttonPressed = false;
const int BUTTON_PIN = 2;
const int LED_PIN = 13;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  pinMode(LED_PIN, OUTPUT);
  attachInterrupt(digitalPinToInterrupt(BUTTON_PIN), 
                  buttonISR, FALLING);
}

void loop() {
  if (buttonPressed) {
    digitalWrite(LED_PIN, !digitalRead(LED_PIN));
    buttonPressed = false;  // Clear flag
  }
}

void buttonISR() {
  buttonPressed = true;  // Set flag only (ISR must be fast)
}
Pro Insight: ISRs should only set flags. Never use Serial.print() or delay() in ISRs.
